Lesson 3: CSS and HTML Document Styling, Formatting and Structure

Introduction: CSS and Styling

CSS is the "styling" of HTML Elements, via a second layer of language (CSS) and aids in the layout when creating a website or HTML page. CSS stands for "Cascading Style Sets", and is literally applied in a cascading type of way.

One may use the classes that are set by the div element within the HTML structure to alter the formatting, colours or anything within the box created by the "div" element, and this can be done through "inline" text editing, placing CSS in the Style section of the HTML or by a reference in the style element to an external CSS document which is a consolidation of the rules the CSS document will apply to the HTML document.

One must pay attention to the formatting when working with CSS. The indentation of the tags is important, or keeping them in line within eachother in a downward way, in order to ensure all relevant tags are closed, and the HTML will run as it is supposed to.

CSS at Work in an HTML Document

CSS will alter the inline styling of elements, or when coded in correctly, will alter the entire layout of a HTML page. One needs to have the correct "div" and "span" elements in the document in order for this to occur.

CSS will "inherit" attributes from previous CSS styles (whether inline or external). One may use inline CSS to alter individual elements of HTML style, and keep the rest of the formatting across the HTML document. According to the Wikipedia entry on CSS and cascading style elements, descendent elements will inherit "ancestor" CSS values while enclosing it.

In general text descendant elements will be inherited, but box related properties will not be. What follows is a list of properties that can be inherited:

  • colour
  • font
  • letter spacing
  • list style
  • text alignment
  • line height
  • visibility
  • white space
  • word spacing

Properties that will not be inherited include:

  • background
  • border and display
  • float and clear elements
  • margin
  • minimum and maximum height
  • width
  • outline
  • overflow elements
CSS Selectors in HTML Documents

CSS Selectors are created within the HTML document, and then using an external CSS Styling Sheet, with references to that "class" by label, one may alter the formatting of those sections as defined by the selector (the selector will apply to all CSS tags using the HTML labels). There are two ways to select elements using CSS:

  • Selection of elements by Class
  • Selection of elements by Tags
How does CSS Control HTML Values?

CSS uses an externalised set of values, defined within an external CSS file, with pre-defined selectors to "control" the attributes within a give selector class within the HTML document itself.

There may be multiple CSS Style Sheets defining different aspects of an indivdual webpage, or to different sections of webpages within an overall document. This enables one to easily modify entire sections of HTML without having to alter individual values in the HTML document itself through the use of inline styling.

One may define selector class values, or other HTML Elements such as Headings, text-formatting and so on in the CSS File.

How to include CSS Styling in a Webpage

Method 1: Write the CSS in an external document and make a link within the Head section of the HTML document. Inside the Head section, on creates a Style tag, and then defines all the CSS within the Style section, this will then apply to all defined elements within the webpage.

Method 2: Link the HTML to an External CSS File. This is better for larger projects, as one can keep the styling and structure of the HTML and CSS more organised. This is done by adding a defining line of code in the Head of the HTML document that refers to the external CSS File by name. This method has been mentioned numerous times in this section, and by far and away seems to be the most prominent way to define and use CSS in HTML.

Method 3: Inline coding of CSS Elements. This method is not as good as the aformentioned methods - as it leads to repeated code (not good coding practice), however it may be used to style individual (standalone) elements that one may not want to "universally" style.

Using the right Elements for Styling

One needs to examine what the best elements are for formatting a HTML document: for instance using a Heading Element, and styling that as oppose to manipulating the text using CSS. The reason for this is simple, HTML readers and browsers "create document structures" based on the layout and heading within a HTML document, and hence with good structure the webpage will be easily located in a web search based on the headings and content, if the page is welll structured.

The Developer Tools (in Chrome) enable one to examine any website and its structure to see how other developers achieve this. One must try and think of styling in terms of the "boxes" and "boxifying" from the previous lesson.

Boxes in HTML and CSS

An example of the box model may be found here, and this is vital for understanding the structure of an HTML documents, and then the CSS formatting of aforementioned boxes.

The Box Model has four layers (each working out from the innermost layer):

  1. Content: the text nucleus, items are essentially "wrapped" and protected in this layer.
  2. Padding: Padding, which clears the area "around" the content.
  3. Border: around the padding.
  4. Margin: transparent, and clears the area around the border.

Each layer "protects data", and can be manipulated individually. This is why formatting can be so difficult as one is trying to balance all of these four elements of boxes to create a layout that satisfies a client. One should note that all calculations of the total width of all the boxes, is the sum of both the sides of the boxes, going left to right across a page.

CSS Box Size:


CSS Box Size includes the Padding and Margin of the box settings in the HTML and CSS styling. One should note that older browsers may struggle with these settings, and if this is the case one should use the code found with the documentation for the older browsers in order to make it work in the older browsers. These are called "browser prefix codes".

As setting the four elements of box size can be tricky, one should use one of the following two techniques: (1) set the box size in percentages (%), or (2) set the "Box-sizing" attribute to "border-box", for each element.

Box Positioning:


Div elements are Block elements, not inline elements, therefore by default they take up the entire width of the webpage (as available). If one wants to make the Div elements "sit next to eachother", one should use the "display: flex" in the relevant CSS Style Code in the CSS file referenced in the HTML file. This will moves the "images" or "boxes" in to alignment, BUT one must ensure the values set for the width of the elements are LESS THAN 100% of the total width of the page (as mentioned above).

Good Coding Practice: Do not repeat Code!

It is good coding practice (in HTML and in general programming) to try and streamline the code. This means one should strive to ensure that code is repeated as little as possible.

In essence this is one of the reasons one uses CSS in relation to HTML, it enables one to set the style of an entire webpage, or sections of it through the use of CSS, which then enables one to alter the style to a client's needs, without affecting the data on the page, and with minimal effort.

This process also makes it easier for other people to read the HTML or Code, without having to read through masses of repeated code which would be an annoying and time consuming process. Breaking a webpage in to sections, with relative CSS styles therefore is very important to this process,

Less repetition of code is more efficient, and by extension makes for programming with less errors.

Code, Test, Refine

In general terms the following process applies to turning a graphic design proposal in to a website:

  1. Boxify the elements on the proposed draft
  2. Examine the Tags, and look for repeated elements or styles which can be used to formulate a potential layout (i.e. repeated styles and semantic elements.
  3. Check the size of the boxes, and create smaller boxes if necessary.
  4. Examine the layout of the HTML/CSS by running it in a Browser, and then refining the elements until one is closer to the desired design.
  5. The Small Details: keep refining and adjusting until the HTML/CSS and page itself have the desired look or design as to the briefing.

One should use the Developer Tools to not only deconstruct the HTML code on a page. but also to "experiment" with altering of values to see what they will look like: this means one may avoid altering the actual HTML/CSS values - and if you make a mistake, you can merely hit Refresh in the browser to reset the values and start again, or if something works - one can copy it over to the HTML/CSS file and save it (one can use the 'Sources' tab in the developer tools section to do this). Compare your design to the proposal, and use Developer as previosuly mentioned to keep refining until you are happy with the look and style of the webpage.
